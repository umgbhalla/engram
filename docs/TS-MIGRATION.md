# TS-MIGRATION — authored source is now TypeScript + Rust

> Migration of all authored runtime/app JavaScript to strict TypeScript, with a
> per-app esbuild build pipeline that emits the `.mjs`/`.js` the Cloudflare runtime
> actually needs. The Rust kernel (`lib.rs` + the rquickjs engine) stays `.rs`.
> Runtime behavior is identical; the build is deterministic.

## Headline

**Yes — the authored codebase is now TypeScript + Rust only.** Every authored
*runtime/app* source file that used to be hand-written JavaScript is now a strict
`.ts` file (or stays `.rs`). The `.mjs`/`.js` that the Cloudflare runtime loads is
**generated by esbuild at build time** from those `.ts` sources (gitignored where
newly cut over). The only hand-written `.mjs` that remain are **Node build/test
glue** (esbuild orchestrators, the engine/wasm build, the SHA-256 engine-hash step,
e2e/test harnesses) — not the kernel/app runtime that this migration targeted — plus
two files deliberately **kept for rollback** until cutover is committed.

## What converted (JS → TS)

| Was (authored JS) | Now (authored TS) |
|---|---|

Support TS added: `tsconfig.json` (strict, `noEmit`), `src/kernel.d.ts` (ambient
module + globals), `src/engine.d.wasm.ts` + `src/stdlib.bundle.d.txt.ts` (sidecars
typing the wrangler asset imports). Rust untouched: `src/lib.rs`,

### apps/cloud (`engram-cloud`) — multi-tenant supervisor

### packages/cli (`@engram/cli`)

`engram.mjs` → `src/engram.ts` (strict TS, `noUncheckedIndexedAccess`). esbuild emits
the runnable `bin/engram.mjs` (shebang preserved exactly once). Aligned to the v2 SDK
`connect({url,session,config,WebSocket})` + `EngramSession` surface. Legacy `rlm` /
`tools` / `--context-file` / trajectory `trace` commands were **removed** — a genuine
backend removal (the kernel no longer dispatches `host.subLM`/`host.ctx`/`host.final`),

### apps/ui (`engram-ui`)

`src/worker.js` → `src/worker.ts`. esbuild bundles with an HTML text loader
(`--loader:.html=text`) into `dist/worker.mjs` (HTML inlined). `src/html.d.ts` declares

authored worker source).

### packages/sdk (`@engram/sdk`)

Already TypeScript before this migration (`src/index.ts` + `.ts` examples) — no JS to
convert; verified, not changed.

## Per-app esbuild build pipeline

The principle: **authored `.ts` (or `.rs`) in, runtime `.mjs`/`.js` out**. Each app
typechecks strict-first, then esbuild (or the Rust toolchain) emits the artifact the
Cloudflare runtime loads. No minify, `format=esm`, `target=es2022` → deterministic.

- **kernel** (`wrangler.jsonc` `build.command`):
  1. `tsc -p tsconfig.json` (strict, `--noEmit` typecheck)
  2. `scripts/build-ts.mjs` (esbuild) → `src/kernel-glue.mjs` (self-contained; `repl-transform` inlined) + `entry.mjs` (wasm/txt/engine-hash/shim external)
  3. `scripts/build-engine.mjs` (cargo `wasm32-wasip1` → `src/engine.wasm`)
  4. `scripts/engine-hash.mjs` (SHA-256 → `src/engine-hash.js`, **after** esbuild so the hash isn't frozen stale)
  5. `worker-build --release` (compiles the Rust DO; reads `src/kernel-glue.mjs` via `#[wasm_bindgen(module=…)]`)

  Invariant: `kernel-glue.mjs` must exist with exact exports
  (`newGlueKernel`/`getEngineHash`/`newMutex`) **before** `worker-build` runs — esbuild
  produces it first. Verified: transform fuzz 4000/4000 + 23/23 hand cases byte-identical
  old↔new; Mutex FIFO preserved; dry-run bundle 2006 KiB / 724 KiB gz.

- **cloud** (`scripts/build.mjs`): `tsc --noEmit` (worker + scripts configs) →
  `bake-rust.ts` (emits `src/modules.rust.gen.js` facet payload) → esbuild
  `src/supervisor.ts` → `dist/worker.mjs` (only `cloudflare:workers` external).
  Determinism: the baked `modules.rust.gen.js` is **byte-identical** between the old
  `.mjs` and new `.ts` baker, so the facet `codeId`/`engineHash` are unchanged (no
  `EngineHashMismatch`). `wrangler.jsonc main` → `dist/worker.mjs`.

- **cli**: `tsc --noEmit` strict → esbuild `src/engram.ts` → `bin/engram.mjs`
  (shebang once, `node --check` OK).

- **ui**: `tsc --noEmit` strict → esbuild `src/worker.ts` (`.html` text loader) →
  `dist/worker.mjs`. `wrangler.jsonc main` → `dist/worker.mjs`; `deploy:ui` builds first.

- **sdk**: `tsc --noEmit` + esbuild + `.d.ts` → `dist/index.mjs` (already TS).

`.gitignore` updated per app to ignore the generated artifacts (`entry.mjs`,
`src/kernel-glue.mjs`, `src/engine.wasm`, `src/engine-hash.js`; `apps/cloud/dist/`,
`modules*.gen.js`; `apps/ui/dist/`).

## Authored-source accounting — ZERO authored raw-JS runtime source remains

**Authored runtime/app source after migration** is `.ts` + `.rs` only:

- TS (authored, working tree): kernel `entry.ts`, `src/{kernel-glue,repl-transform,stdlib-meta}.ts` + 3 sidecar `.d.ts`; cloud `src/supervisor.ts` + `scripts/bake-rust.ts` + `modules.rust.gen.d.ts`; ui `src/worker.ts` + `html.d.ts`; cli `src/engram.ts`; sdk `src/index.ts` + 4 example `.ts`.
- Rust (authored): `apps/kernel/src/lib.rs` (DO shell) + `apps/kernel/engine/src/lib.rs` (rquickjs engine driver). Everything else under `engine/vendor/rquickjs/**` is **vendored**, not authored.

The `.mjs`/`.js` the runtime loads (`entry.mjs`, `kernel-glue.mjs`, `dist/worker.mjs`,
`bin/engram.mjs`, `modules.rust.gen.js`) are **build outputs** (esbuild/baker), now
gitignored where freshly cut over.

### Honest residue (not kernel/app runtime, intentionally not converted)

1. **Node build/test glue** stays `.mjs` (it runs on Node at build/CI time, is not the
   worker runtime this migration targeted): kernel `scripts/{build-ts,build-engine,engine-hash}.mjs`;
   cloud `scripts/{build,build-stdlib}.mjs`; root `scripts/e2e-ui.mjs`; and the whole
   `tests/**/*.mjs` harness set (out of app/lib code by repo convention). Converting these
   adds no runtime value; they typecheck under `allowJs` where wired. Follow-up optional.
2. **Kept for rollback** until the cutover commit lands (the no-commit guardrail means the
   old files are still in the git index even though the working tree has switched): cloud
   `src/supervisor-rust.js` + `scripts/bake-rust.mjs`. These are no longer wired into any
   build (`wrangler.jsonc main` points at `dist/worker.mjs`).
3. **`apps/cloud/src/modules.gen.js`** is a stale legacy generated blob (`// GENERATED …
   do not edit`), already gitignored — not authored.

### Git-index caveat (no-commit guardrail)

Because nothing was committed, `git ls-files` still lists the deleted JS
(`repl-transform.mjs`, `stdlib-meta.js`, `ui/src/worker.js`, `cli/engram.mjs`) and the
two newly-gitignored kernel build outputs (`entry.mjs`, `kernel-glue.mjs`) as
tracked-but-modified/deleted. The **working tree** is already TS+Rust. The cutover
commit must `git add` the deletions and `git rm --cached entry.mjs src/kernel-glue.mjs`
(they're now `.gitignore`d build outputs) to make the index match.

## Deploy-phase gotchas (carry forward)

- The committed `apps/cloud/wrangler.jsonc` declares the loader as top-level
  `"worker_loaders": [{ "binding": "LOADER" }]`. **wrangler 4.97 silently drops that key**
  → `env.LOADER` undefined → every `/create`+`/eval` fails. Before a fresh live cloud
  redeploy, switch to the `unsafe.bindings` form (`{ "name": "LOADER", "type":
  "worker_loader" }`) or pin a wrangler that accepts the top-level key. The live worker
  already has the binding from a prior compatible build.
- Repo-local `npx wrangler` resolves to stale `3.107.2` (errors on `worker_loaders`); use
  a 4.x wrangler for cloud ops.
- `/admin/keys` and `/eval` read params from the **query string**, not the JSON body.
