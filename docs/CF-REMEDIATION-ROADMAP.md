# ENGRAM REMEDIATION ROADMAP

*Canonical. Every effort/feasibility below reflects the adversarial VERIFY corrections, validated against live code (`set_clock` absent; `__rand` is a Rust closure; host-call boundary is async-park-only; `FsStageOp` ∈ {Write, Delete}; `registry_invoke` is frame-level; 24MB guard at kernel-glue.ts:2581).*

## 1. Headline

Of the 89 unacceptable + 3 dangerous findings, **~85 are fixable and ~7 stay hard-limited** — but the means is more conservative than the per-theme self-assessments claimed. The dominant lever is **pure in-VM / host-side JS** (console, buffer encodings, util.inspect/TextDecoder, streams, fetch/http fidelity, fs-tree in-heap semantics, TS reject-gate, fetch-cap typed-reject, process.env, vm same-realm hardening) — these need **no CF primitive and no determinism change**, and clear the largest block of findings. A second tier is **wiring already-bound CF primitives**: Worker Loader (`LOADER.load`) for *real* VM isolation, DO Alarms (supervisor-owned, since facets can't alarm) for cross-hibernation timers, R2 streaming GET→PUT for atomic large-file rename, and the Tier-2 container as the escape hatch for real-Node TLS/mTLS and full-ICU. A third tier is the **determinism-vs-realism axis** (`config.realtime`): unifying ~15 findings, but **NOT the "S, glue-only" job the menu claimed** — real entropy must be plumbed through the Rust `__rand` closure (an engine edit + rebuild), and `set_clock` *does not exist in the shipped engine* (the "clock:real already works" premise is false). **Three findings are genuinely immovable** (the snapshot incompressible cliff's in-kernel ceiling, raw TCP keepalive/NODELAY, in-VM TLS cert-chain/ALPN/mTLS) — these become document-only honest-signal fixes plus a container escape hatch. **Zero new WASM imports** are needed anywhere; the 12-import ceiling is untouched by every fix.

## 4. `config.realtime` — the unified realism switch

A single coherent two-axis switch replacing the scattered partial flags. **All OFF by default; seeded byte-identical restore stays the default contract.**

```
config.realtime: true              // umbrella → clock + crypto + timers all real
config.realtime: { clock, crypto, timers, mtime }   // granular override
```

**What it enables:**
| Axis | Real behavior | Implementation reality (corrected) |
|---|---|---|
| **clock** | `Date.now`/`performance.now`/`hrtime`/`uptime`/`dayjs.now`/`mtime` track real wall time; real inter-cell elapsed | ⚠️ **`set_clock` does NOT exist in the shipped engine** (kernel-glue.ts:1571 try/catch silently no-ops). Today clock:real only sets a create-time base then drifts +1ms/call. **Real fix = build the `set_clock` export** (engine edit + rebuild). The menu's "already shipped" is false. |
| **crypto/entropy** | `Math.random` + node:crypto + nanoid/uuid + `crypto.getRandomValues` → true workerd CSPRNG | ⚠️ **NOT glue-only.** `__rand` is a Rust closure inside WASM (lib.rs:2932); every user-facing consumer routes through it, NOT through WASI `random_get`. Swapping `random_get` alone achieves nothing visible. Real fix = gate `__rand` by a `cryptoReal` flag and have it read real bytes (per-turn-refilled scratch ring written by glue via `crypto.getRandomValues`, chunked ≤64KB/quota). **Unbounded-draw problem unsolved**: a `Math.random()`-heavy loop exhausts any fixed per-turn buffer with no mid-cell host re-entry (sequential host-call boundary). Needs a new `set_entropy_mode` export. |
| **timers** | `setTimeout`/`AbortSignal.timeout`/socket idle fire on real wall time (TIER A, single-digit-sec cap) | §3 — own budget bucket, pins kernel resident |
| **mtime** | fs timestamps from real clock | rides the clock axis + fs_files column (§3) |

**EXACTLY what it costs:**
- **Byte-identical restore is forfeited** for the enabled axes — by design, identical trade to today's `clock:"real"`. Two runs of the same source from the same session diverge in the heap.
- **Replay/comparison becomes meaningless** for that session — but the kernel *never replays* (heap-snapshot, no journal), so this costs **nothing operationally** except: engine-migration replay (the oplog path on engine-hash change) can no longer reproduce identical bytes for a realtime session.
- **Snapshot/restore mechanism unaffected** — still an ArrayBuffer blit; flags persist in `config_json` (same path as rngSeed/clock), re-armed on restore.
- **Observability:** add an AE `configRealism` blob (`none`/`clock`/`crypto`/`timers`/`all`) per op so determinism status is queryable.

**Honest effort:** the menu said **S/ship-now**; it is **M with an engine rebuild** — both real axes (clock via new `set_clock`, crypto via `__rand` reroute + new `set_entropy_mode`) are Rust engine work, plus a re-test of seeded byte-identical restore in the default path. The CF *primitive* (`globalThis.crypto.getRandomValues`, `Date.now`) is trivially available (no binding, sync, no quota beyond 64KB/call) — that was never the hard part.

**Findings unified (~15):** all crypto-entropy items, all wall-clock-timer items, Date/performance/hrtime/uptime non-realism, mtime=0, dayjs.now, and the determinism-policy meta-finding.

## 5. The snapshot cliff (the dragon)

**Best achievable mitigation = three honest layers; the in-kernel ceiling is immovable.**

- **Layer 1 — IN-KERNEL CEILING IS INFEASIBLE (hard limit, document it).** WASM linear memory is monotonic; dlmalloc does no downward compaction. The OOM is the **dump's transient resident expansion** — `new Uint8Array(memory.buffer.slice(0))` (full 2nd copy) + zstd (3rd buffer) inside the ~128MB DO RAM budget — **NOT the R2 write**. Therefore R2 multipart/streaming, D1, KV offload of the *stored image* **cannot raise the in-kernel ceiling**. True in-place compaction needs value-serialization that loses promise/closure/exotic-namespace fidelity (the whole thesis). **~24MB incompressible is the immovable in-heap ceiling.**

- **Layer 2 — ALREADY FAIL-SAFE for the snapshot path; harden the residue.** The `INCOMPRESSIBLE_BUFFER_CEILING_BYTES=24MB` guard (kernel-glue.ts:2581) **already throws a typed `SizeAdmissionError` before the slice/compress** — socket alive, reset recovers. So **yes, the snapshot path is a typed reject, NOT WS-1006.** The genuinely uncatchable residue is narrow: a single synchronous **native-builtin alloc burst** that crosses ~28MB before the cell yields. Hardening: clamp `ArrayBuffer`/`TypedArray`/`Buffer.alloc/.fill` length args in injected JS (precedent: scryptSync clamp engine:1839) to throw `RangeError` pre-growth. **Honest claim downgrade:** this catches the **canonical single-giant-alloc** (`new Uint8Array(40MB)`) but is **bypassed** by `String.prototype.repeat`, `Array(n).fill().join`, `ArrayBuffer.prototype.resize/transfer`, `%TypedArray%`/`Reflect.construct` subclassing, huge-string `JSON.parse` — rquickjs has **no native alloc-size hook**, so it's a global-constructor shadow (fragile, long tail remains). The post-cell growth tripwire **already exists** (engine finalize_cell:3243). So: "closes the residue" is **too strong → "clamps the common case, residue remains."**

- **Layer 3 — RAISE EFFECTIVE WORKING-SET via offload (the real ceiling-raiser, opt-in).** For legit >24MB working data, don't hold it in the QuickJS heap — push to the **already-bound Tier-2 container** (256MB–20GB, shares R2 /workspace) or R2 directly under `fs/<doId>/offload/`, keep only handles/cursors in the snapshottable heap. Thin `host.offload(key,blob)`/`host.fetchOffloaded(key)` over existing `__hostCall` + R2/sandbox fetch. **Zero new imports.** The snapshot becomes a small index over large external state → big-working-set sessions snapshot/restore fast and never approach the cliff. R2-only offload is deterministic; container offload is opt-in non-deterministic.

**Honest ceiling:** in-heap live state stays bounded at ~24MB forever. **Can it be made fail-safe? — YES, already is, for the snapshot path** (typed `SizeAdmissionError`). The only WS-1006 residue is the narrow native-alloc burst, partially (not fully) closed by Layer 2 clamps.

## 6. Hard limits that stay (document-only)

No CF primitive fixes these:

1. **Snapshot in-heap ~24MB incompressible ceiling** — WASM monotonic memory + dump transient expansion. (Mitigate via offload §5-L3; cannot raise in-kernel.)
2. **Raw TCP keepalive / `setNoDelay` (TCP_NODELAY)** — `cloudflare:sockets` `SocketOptions` exposes only `{secureTransport, allowHalfOpen}`. No knob exists. (No-op returning `this`; Node tolerates.)
3. **In-VM TLS introspection / mTLS** — `getPeerCertificate`/`getProtocol`/`getCipher`/`alpnProtocol`/`authorized`/client-certs/`rejectUnauthorized:false`. TLS terminates inside workerd; neither host JS nor VM ever sees the chain. (Honest-signal: null/{}/false; **container escape** for real mTLS/ALPN/pinning.)
4. **Full ICU legacy-multibyte decoders** (~100: shift_jis/gbk/big5/euc) — multi-MB ICU tables would push images toward the cliff. (Container escape; single-byte + UTF-16 done in-VM.)
5. **Synchronous `existsSync`/`require('./x')` over R2-only files** — sync-over-async is structurally impossible (explicitly not-built). Async `access`/`use('./x')` only.
6. **`import()` native syntax through a parking loader (vm-isolation c1 / module-system c1)** — QuickJS module resolution inside `Module::eval` is **synchronous**; the host-call boundary is a full async unwind. A `set_loader` Loader **cannot park/await** (proven by `__esmEval`'s `WouldBlock`). **Redesign required** (pre-resolve all imports before eval, or a sync in-heap module-source cache) — the parked-loader design is reject-as-described, not "follow-up."
7. **Facets cannot set alarms** — TIER-B cross-hibernation timer scheduling must live on the supervisor DO, not the facet.
