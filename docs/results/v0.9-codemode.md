**Host tools exposed:** `host.ctx.{len,slice,grep,chunk,get,names}`,
`host.subLM`, `host.final`, `host.finalVar`.

chunk-all strategy overruns the replay-shim pump budget at 1091 chunks. A
grep-first / smaller-context model handles full size fine. This is a default-
model strategy limit, not a `host.ctx`/kernel defect.

## 2. The durable advantage â€” pause/resume a code-mode / RLM session

This is the headline differentiator: **you can hibernate a live code-mode/RLM
session and resume it cold.**

- `sessions list` shows recorded repl+rlm sessions (id/kind/ts/endpoint);
  `sessions inspect` returns live generation/inMemory/engineHash/committedCell;
  `trace <id>` dumps generation + committedCell + final.
- **Durable hibernate/resume mid-session (PASS at sub-limit context):** with a
  ~217KB context, set partial RLM state â†’ `hibernate()` (evict,
  inMemoryâ†’false) â†’ reconnect â†’ restored from SQLite snapshot at committedCell
  1; in-VM `partial` continued to `["step1","step2"]` and host-side context was
  still grep-able (needle found after resume).
- Smoke evidence: after evict (`inMemory:false`) the context handle
  (len=1,050,032) **and** namespace (marker===777) survive restore. The context
  store + KV travel in the snapshot manifest meta (new `ctx_json` column,
  best-effort ALTER migration).

**Honest limit:** `trace` on a fresh reconnect shows `final:null` â€” the final
answer lives in transient per-live-session host state (delivered correctly
in-band during the `rlm()` run) and is not persisted in the snapshot for later
trace.

## 4. Bugs / risks (ranked)

| # | Severity | Issue | Impact / Fix |
|---|----------|-------|--------------|
| 1 | **HIGH** | **`setContext` persists the whole blob as a single SQLite manifest row â†’ `SQLITE_TOOBIG` on large contexts.** The 4.36MB context failed: `"manifest insert ... string or blob too big: SQLITE_TOOBIG"`. On failure `committedCell` never advances, so a large context (and subsequent eval state) does **NOT** survive eviction. | Durable-hibernation claim holds only for contexts under the ~1MB SQLite per-value limit. | 2 | MED | **Default chunk-all RLM root model exhausts the pump budget** at large chunk counts (1091 chunks â†’ stops at 45 subLM calls). | Not a kernel bug; ship a grep-first default rootModel or raise/communicate the pump budget. |
| 3 | LOW | **`trace` shows `final:null` post-reconnect** â€” final answer is transient per-live-session, not snapshotted. | Persist the recorded final into the snapshot manifest. |
| 4 | LOW (doc) | **Async-IIFE object completion value previews as `{}`** (existing kernel unwrap limitation). | SDK already works around it: `execute()` JSON-parses `value`/`valuePreview`; RLM reads the subLM queue + final out-of-band. Prefer sync returns or `host.final`. |
| 5 | LOW (doc) | **`registerTool` local-HTTP bridge** (`_ensureBridge`, 127.0.0.1) only reachable when SDK + kernel are co-located. | 6 | TRIVIAL | `host.ctx.slice`/`get` capped at 256KB per call (avoids recoverable WS-1006 from oversized native alloc). Cargo package name still `montydyn-v05` (cosmetic). Global wrangler 3.107 so deploy used `npx wrangler@4`. | Known/cosmetic. |
