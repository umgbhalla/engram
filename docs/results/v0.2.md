## 1. Are BOTH gates now GREEN? (skeptical, with the decisive numbers)

### P0 — cold-restore-after-spike: **YES, decisively GREEN.**

This is the unambiguous win. The wedge was: WASM linear memory is monotonic, so a session that ever spiked its buffer past the old 20MB `MAX_RAW_BYTES` raw ceiling, then freed + checkpointed (warm-fine, tiny used heap), could **never cold-restore** — every restore threw `SizeAdmissionError`, escape was `reset` (total state loss).

**Fix:** the restore guard now admits on the snapshot's recorded `usedHeap` (new `used_heap` manifest column, wired `lib.rs` → `restore()`), not the raw image bytes. A separate raw ceiling (`MAX_RESTORE_RAW_BYTES`=45MB, pinned to dump-side `MAX_DUMP_BUFFER_BYTES`) is kept only so a genuinely-too-big image still fails safe.

**Decisive numbers (live, fresh DO, cold restore after spike→free→checkpoint→evict→eval):**

| Peak | spike raw image | freed usedHeap | RESTORE | restoreSource | SizeAdmission |
|------|------|------|------|------|------|
| 15MB | 17.04MB | 0.08MB | `ok=true [12345,"intact"]` | sqlite-restore | none — **PASS** |
| 22MB | 24.38MB | 0.08MB | `ok=true [12345,"intact"]` | sqlite-restore | none — **PASS** |
| 24MB (the wedge) | **26.5MB** (>20MB old ceiling) | ~0.08MB | `ok=true value=777` | sqlite-restore | none — **PASS** |
| 35MB | 38.01MB | 0.08MB | `ok=true [12345,"intact"]` | sqlite-restore | none — **PASS** |
| 60MB never-freed | 64.2MB buffer | — (live) | refused at dump | — | typed `SizeAdmissionError` (fail-safe, socket alive) — **PASS** |

The decisive sub-case (`>20MB raw → free → evict → cold restore SUCCEEDS, value=777`) passes; the genuinely-huge live image still fails SAFE with a typed error and a live socket. P0 is closed.

> Honesty note: one gate run initially reported 2 FAILs — that was a throwaway test-harness bug (asserting `r.value` as a native array when the value arrives via `valuePreview`). Corrected assertion → 4/4 PASS. The underlying kernel behavior was always correct.

### P1 — `while(true){x=1}` → TimeoutError, socket-alive: **GREEN on the decisive sub-case, but FRAGILE.**

The decisive sub-case passes: `while(true){x=1}` trips a typed `{ok:false,error:TimeoutError}` in **~333–864ms** with the socket **OPEN (`readyState=1`)** and the **next eval working (`value=42`, re-confirmed `value=4`)**. The easy sub-cases (empty `while(true){}` ~178–361ms; `while(true){globalThis.x=1}` ~403–1034ms; `let o={};while(true){o.a=1}` ~389–544ms) all trip clean. Safety sampling: 30/30 and 40/40 infinite-loop trips clean, 0 WS-1006, 0 next-eval failures. Smoke 52/52.

**Why "fragile," not "robustly GREEN":** the decisive `x=1` trip is NOT 100% at every candidate budget. workerd throttles the host interrupt callback after a load-sensitive number of invocations per synchronous turn; if the tick budget sits above that floor, the heavier property-store loop shapes (`globalThis.x=1`, `o.a=1`) can ride past the handler to the DO wall → WS 1006. Measured fresh-DO escape rates (load-sensitive, non-deterministic):

| budget | `x=1` trip | 10M legit loop completes |
|--------|------------|--------------------------|
| 4000 | escapes ~1/3 (WS 1006) — **UNSAFE** | 4/4 |
| 3000 | 4/4 trip | 3/3 |
| 2500 | ~7/8 trip | 3/3–4/4 |
| 2000 | 4/4 trip | **0/3 (false-trips 10M)** |
| 1500 | ~5/6 trip | 0/5 |
| 1200 | 0/18 escape per shape (100% trip) | 0/5 |

The two P1 requirements (every infinite loop trips socket-alive AND a legit 10M loop completes) are in **direct tension** and no single static budget satisfies both 100%/100% on this workerd/QuickJS-ng build, because `Date.now()` is frozen during sync exec (no wall-clock fallback) and quickjs-wasi 3.0.0 exposes no interrupt-interval control. The decisive safety property (`x=1` trips socket-alive) is met at the deployed budget; the 10M completion is the casualty at the lower end.

**P1 verdict: GREEN on the non-negotiable safety axis. Distinguish: the easy sub-cases pass robustly; the decisive `x=1` sub-case passes but only because the budget is tuned to a narrow, jittery window — it is correct-but-brittle, not bulletproof.**
