# V1.1 — Multi-tenant facet at v0.8 feature-parity + adaptive keep-warm

## Scope

V1.1 closes the V1.0 feature-parity gap inside the multi-tenant facet (the v0.8 kernel's stdlib + Tier-0 extensions + mediated fetch were not yet wired into the facet runtime), and adds adaptive keep-warm in the SupervisorDO.

## Build summary

- Copied v1/ -> v1.1/, deployed montydyn-v11 (version 5b1ef79a).
- Stdlib: `build-stdlib.mjs` esbuilds lodash/dayjs/nanoid/uuid/zod (+ mathjs opt-in), baked into the facet as a `{text}` loader module, evaled into the heap at create and snapshot-persisted. 500KB source cap + mathjs-opt-in admission guards live.
- Extensions: 5 Tier-0 quickjs-wasi `.so` (crypto / encoding / url / structured-clone / headers) vendored as `.wasm`, delivered as `{wasm}` loader modules, wired at create AND restore via `globalThis.__QJS_EXT_MODULES`.
- Mediated fetch: `globalOutbound` changed null -> per-tenant `HttpGateway` WorkerEntrypoint (`ctx.exports.HttpGateway`); in-VM `config.fetch` allowlist now reaches the network. `MEDIATED_FETCH_ALLOW/DENY` coarse env backstop.
- Adaptive keep-warm: SupervisorDO sessions SQLite + cadence columns (eval_count, inter_arrival_ewma_ms, warm_until, latency_sensitive, last_size_gz); `_touch` computes EWMA, `_shouldWarm` decides, a 30s alarm sweep heartbeats predicted-active facets (`facet.ping()` RPC) and evicts idle. Routes: `/stdlib`, `/warm`, `/configure?latencySensitive=1`.
- Fixed an ms-timestamp `|0` truncation bug that zeroed the keep-warm EWMA (now `Number(x)||0`).
