## What shipped
LAMBDA-RLM bounded combinators + durable multi-turn agent code-mode adapter.

- **LAMBDA-RLM combinators** (`v0.9.2/stdlib-src/lambda.js`, ~3.5KB IIFE, default bundle): `SPLIT(ctx,by)`, `MAP(parts,leafFn)` (parallel leaves), `REDUCE(results,mergeFn)`, and `lambdaRLM(query,opts)` bounded recursion driver (fix/Y-combinator). Pure in-VM JS composing `host.ctx.*`/`host.subLM`; no new VM/host primitive; snapshot-captured so it survives hibernation. Exposed as `globalThis.lambda.*` and SDK `session.lambdaRLM(...)`.
- **Termination guarantee**: every recursive descent decrements `maxDepth`; every leaf decrements `costBudget` (hard cap on total `host.subLM` calls). Over-decomposition is bounded, not exponential.
- **Agent code-mode adapter** (SDK): `Agent` class + `createAgent({endpoint,id,config,tools,onSubLM})`; `agent.turn(code) -> {result,logs,toolCalls}`; `registerTool` maps a tool -> `host.<name>` -> `host_call` via a cooperative replay shim (no inbound connection). `agent.hibernate()/resume()` make multi-turn state durable in the snapshot. Agent tool names shadow demo builtins; `host.lambda` stays reachable.
- SDK bumped to 0.9.2 with `index.mjs` + `index.d.ts` types. Worker renamed `montydyn-v092`; R2 keys namespaced `v092/` on shared `montydyn-snapshots`.

## Known limitations / handoff
- Leaf oracle and agent tools run **client-side via a cooperative replay shim** (queue + re-run), NOT an inbound HTTP bridge — a remote worker drives them with no reverse connectivity. Cost: loop caches one new leaf/tool result per re-run, so N distinct leaf prompts need ~N pumps; raise `maxPumps` for large fan-outs. `costBudget` is the always-correct hard invariant; the `exhausted` flag is best-effort (identical leaf prompts dedup by key).
- Per-session state hygiene: `lambdaRLM` clears `__subLM_answers/__subLM_queue/__lambda_result` at start of each call (fixed an early flake when two calls shared a session).
- Async-IIFE cell values can preview as `{}` (known kernel unwrap limit); agent turn reads `__agent_lastResult` to recover the structured result.
- Hibernation is between cells/turns, not mid-flight during a sub-LM fetch (unchanged from v0.9.x); the agent turn boundary is exactly the supported hibernation point.
- `README.md` in the v0.9.2 dir is stale (titled v0.3) — cosmetic.
- Reused R2 `montydyn-snapshots`; no resources deleted; nothing published.
