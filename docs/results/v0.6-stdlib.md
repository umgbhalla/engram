# v0.6 — Configurable In-VM Stdlib (canonical study)

> Worker `montydyn-v06` (https://montydyn-v06.umg-bhalla88.workers.dev).
> Branch `feat/v0.6`. AE dataset `montydyn_kernel`, R2 prefix `v06/`. 2026-06-01.
> Detailed sweeps: `docs/results/v0.6-stdlib-size-sweep.md` (size + cost + compat axes).

## What v0.6 adds

A configurable in-VM standard library. Curated, pure-JS libs are esbuilt at build time
(`scripts/build-stdlib.mjs`, `platform=browser` so any Node-builtin dependence fails the
build) into one Text module (`src/stdlib.bundle.txt`) holding `{moduleName: iifeString}`.
wrangler ships it as a `Text` module; `entry.mjs` sets `globalThis.__STDLIB_BUNDLE`. At
`create`, `config.modules:[...]` selects a subset and `glue.createFresh()` evals only those
IIFE strings into the live QuickJS global namespace **after `REBIND_SRC`** (which now also
creates a seeded `globalThis.crypto`, required by nanoid/uuid). The injected globals are
ordinary heap objects, so the existing heap-snapshot `dump()` captures them and `restore()`
blits them back — **they persist across hibernation with no re-inject path**.

Default stdlib: `lodash-es`, `dayjs`, `nanoid` (+`customAlphabet`), `uuid` (v4/v5/validate),
`zod`. Default bundle ~175 KB source; full 14-lib battery ~1.25 MB source.

# Convergence + Breaking Point

## 1. Does configurable in-VM stdlib injection WORK + survive hibernation? YES.

Proven live on `montydyn-v06`:

- **Inject + run.** `create {modules:true}` -> `stdlib.loaded=[lodash,dayjs,nanoid,uuid,zod]`;
  all run correctly: `_.chunk([1..5],2)=[[1,2],[3,4],[5]]`, `dayjs.format=2020-06-15`,
  `nanoid().length=21`, `uuid.validate(uuid.v4())=true`, `zod.safeParse=true`.
- **Survives real hibernation.** Set `userState=600`, evict + 75 s idle (real hibernation) ->
  reconnect: `generation 1->2`, `inMemory=false`, `restoreSource=sqlite-restore`. All 5 libs
  STILL work, `userState=600` survived, and the reconnect `create` reply shows `stdlib.loaded=[]`
  — proving the snapshot carried the libs with **no re-inject**.
- **Subset selection** works and survives restore (`{modules:[lodash,dayjs]}` -> `z` undefined,
  persists across evict-restore). `{t:stdlib}` introspection returns catalog + versions.
- **v0.5 regression clean**: no-modules session `x=42`/`inc()=43` via sqlite-restore PASS.
- **All 14 libs inject + run** in QuickJS-ng (esbuild IIFE -> `globalThis`) once the seeded
  `globalThis.crypto` shim is present. The crypto shim is deterministic (seeded host entropy),
  a strict addition to `REBIND`, ordering: stdlib injected **after** REBIND.

## 2. THE BREAKING POINTS (numeric)

The binding constraint is the **per-session snapshot checkpoint of the live, incompressible
heap** — NOT source bytes, NOT the worker bundle cap, NOT parse cost.

### a. Max stdlib size within the safe envelope

| Metric | Safe ceiling | Source-side equivalent |
|---|---|---|
| Snapshot raw image | ~20 MB safe / ~43 MB last-good synthetic compressible | — |
| WASM linear-memory buffer | crosses danger at ~24 MB | ~0.75 MB src (mathjs alone) |
| Used heap | ~5–7 MB for a snappy session | ~507 KB src (13-lib mid set) |
| Snapshot gz (R2 overflow >2 MB gz) | ~0.95 MB bundle | — |
| **Recommended hard cap** | **total injected source ≤ ~500 KB** (≈ ≤7 MB raw / ≤2 MB gz / ≤~5–7 MB used heap) | 13-lib set |

**Two distinct death modes — note the dangerous gap between them:**

1. **Uncatchable OOM (WS 1006 / Error 1102, DO killed)** when the live heap pushes the WASM
   buffer to only **~24–30 MB raw** (image ~24–32 MB). This is BELOW the 45 MB
   `MAX_DUMP_BUFFER_BYTES` guard, so the dump is *admitted* and `snapshot()+serialize`'s
   ~2–3x transient copy (~60–90 MB) OOMs uncatchably. `mathjs` alone (746 KB src, buffer
   ~24 MB) and the full 14-lib set (1.25 MB src, raw 29.9 MB / used 14.2 MB / buffer 24.6 MB)
   both trip this. All libs inject and run fine; death is purely at the **first checkpoint**.
2. **Clean typed failure (safe)** when the buffer exceeds **~45 MB**: typed `SizeAdmissionError`,
   socket alive, no checkpoint, no OOM. Synthetic 48 MB & 56 MB retained heap clean-reject;
   evict then cold-restores "fresh".

A **third, separate, lower cliff**: a single inline `config.stdlib` source blob
**≥ ~2.1 MB** WS-1006s at create (one-shot compile transient of a multi-MB string).
The production per-module path (`config.modules`, eval each lib separately) avoids it.

### b. Cold-wake degradation curve (gentle, linear, no cliff, all sub-second)

| Raw snapshot | cold-wake |
|---|---|
| 2.6 MB (small/5-lib) | 227 ms |
| 5.0 MB (13-lib mid) | 357–494 ms |
| 9.7 MB | 263 ms |
| 22.3 MB | 456 ms |
| 34.9 MB | 546 ms |
| 43.3 MB (last good) | 633 ms |

Persistence is **free**: libs ride in the restored heap, not re-injected. A 13-lib stdlib
survives full evict + cold-wake in ~357 ms.

### c. wrangler bundle-cap interaction — NOT binding

Full 14-lib stdlib embedded: **3150 KiB upload / 1054 KiB gzip**, ~13x headroom under the
~10 MB compressed cap (~9 MB free). The Text module grows the worker 1:1 with bundle source,
but the per-session snapshot ceiling is reached far sooner. Default 175 KB bundle deploys at
760 KB gz.

### d. eval/parse cost ceiling — cheap, linear, paid ONCE at create

~80–90 ms per MB of bundle source (29 ms @175 KB -> 540 ms @6.65 MB; local ~11.5 ms/MB).
Source->heap amplifies ~12–18x for typical libs, **29x for mathjs** (729 KB -> 20.6 MB heap).
Create round-trip vs nLibs: 0->166 ms, 5->217 ms, 8->279 ms, 13->517 ms, mathjs-alone->1560 ms,
all-14->2830 ms-then-killed. Per-cell checkpoint (the real recurring cost) scales with Σ lib
heap: 0 libs ~150 ms, 13 libs ~450 ms, mathjs ~4400 ms. The 1200-tick per-cell budget is NOT
the constraint (injection runs budget-disabled).

## 3. Recommended default stdlib + hard limits to enforce

**Recommended defaults (all <1 MB heap each, SQLite-store, snappy):**
`lodash-es, dayjs, date-fns, ramda, immer, decimal.js, nanoid, uuid, zod, js-yaml, papaparse, marked`
(rxjs ok to include). This 13-lib set: create ≤520 ms, per-cell ≤450 ms, cold-wake ≤360 ms,
raw ≤5 MB, all SQLite.

**Hard limits to enforce in the config guard (the key gap to close):**
1. **Reject mathjs-class eager mega-libs from defaults** — opt-in only.
2. **Add a config guard on total injected source size: cap ≤ ~500 KB** (proxy for ≤7 MB raw /
   ≤5–7 MB used heap), rejecting at create with a typed error.
3. **Lower `MAX_DUMP_BUFFER_BYTES` to ~18–20 MB** so heavy bundles clean-reject
   (`SizeAdmissionError`, socket alive) instead of crossing the silent 24–30 MB OOM cliff.
   This is the single most important fix: it converts the dangerous uncatchable death into a
   safe typed rejection.
4. **Disallow the inline single-blob `config.stdlib` path ≥2 MB** (or route it through the
   per-module eval path).

Recoverable today: a too-big stdlib does NOT brick the session — reset + fresh create with a
safe set works (verified).

## 4. GO / NO-GO

**GO — viable-with-limits.** Configurable in-VM stdlib injection works, runs, and survives
real hibernation for free across the entire 13-lib safe set, with sub-second cold-wake and
~9 MB of worker-bundle headroom. The one breaking point (uncatchable OOM at ~24–30 MB raw)
is well outside the recommended envelope, is reachable only via mega-libs or unbounded blobs,
and is mitigable by a source-size config guard + lowering `MAX_DUMP_BUFFER_BYTES` to make it a
clean typed rejection. Default 175 KB bundle is far inside all three ceilings (size, gz, buffer).

**Conditions on GO:** ship the three guards above (size cap, mathjs opt-in, lowered dump
ceiling) before/with test-all; treat mathjs as the explicit out-of-envelope case in the suite.

## 5. What "test all" must cover on v0.6

1. **Inject + run** each safe-default lib (representative call each) — 13/13 PASS.
2. **Subset selection**: `{modules:[...]}` injects exactly the named libs; unselected globals
   undefined.
3. **Hibernation persistence**: create with libs -> evict + idle past hibernation -> reconnect
   -> all libs still callable, `restoreSource=sqlite-restore`, reconnect `stdlib.loaded=[]`
   (no re-inject), user state survives, generation increments.
4. **crypto shim determinism**: nanoid/uuid produce valid output AND deterministic under the
   seeded entropy; REBIND ordering (stdlib after crypto) holds.
5. **Size/OOM guard**: total injected source over the cap -> typed rejection at create (socket
   alive), NOT WS 1006. Verify the lowered `MAX_DUMP_BUFFER_BYTES` clean-rejects mathjs and the
   all-14 set. Verify session recovers (reset + fresh safe create).
6. **Inline `config.stdlib` blob**: ≤2 MB OK; ≥2.1 MB rejected cleanly (not WS-1006/timeout).
7. **Cold-wake curve**: assert sub-second wake across the 2.6–5 MB raw range; assert SQLite
   store (no spurious R2 overflow) for the safe set.
8. **Cost ceilings**: create ≤520 ms and per-cell checkpoint ≤450 ms for the 13-lib set.
9. **Bundle cap**: deployed worker gz under 10 MB cap with the shipped default bundle.
10. **v0.5 regression**: no-modules sessions unaffected (`x=42`/`inc()` via sqlite-restore).
11. **Introspection**: `{t:stdlib}` returns catalog + versions; `stdlib.loaded` accurate at create.
