## P3.1 — Real outbound fetch egress + allowlist enforcement

### What was inert before
In V0.1/V0.2 the `config.fetch` allowlist was plumbed through `normalizeConfig` but no
host tool ever performed a real outbound request — `fetch` was dead config.

### What V0.3 does
`host.fetch(url, init)` performs a **real DO-side `fetch()`** and resolves to a structured
object `{status, ok, headers, body}` where `body` is response text.

- **The async boundary.** quickjs-wasi host callbacks are *synchronous* (must return a
  value immediately), but a real fetch is async. The kernel's eval was synchronous. So eval
  is now **ASYNC**:
  - `evalCode` runs the cell with `EvalFlags.ASYNC` (top-level `await` allowed), stores the
    returned top-level promise to a VM global `__cellP`, then **pumps**: drain VM microtasks
    (`executePendingJobs`), await any in-flight host fetches, drain again, until
    `__cellP.promiseState` settles, then `resolvePromise(__cellP)` for the final value.
  - `host.fetch` is wired through a dedicated `__hostFetch(url, initJson)` host callback that
    *synchronously* creates a VM Promise (`vm.newPromise()`), kicks off the real `fetch()` on
    the host event loop, registers the in-flight work in `ctx.pendingFetches`, and returns the
    deferred handle. When the fetch settles, it resolves the deferred with a JSON string
    `{status, ok, headers, body}`; the VM-side `host.fetch` wrapper `JSON.parse`s it into a
    structured object for the cell.
  - **Deadlock avoided:** the pending entry awaits only the fetch *work* (which calls
    `deferred.resolve`), NOT `deferred.settled` — `settled` only fires after the *next*
    `executePendingJobs`, which the pump runs after the await; awaiting it in-line would
    deadlock (found and fixed during bring-up).
  - `lib.rs`: the `evalCode` binding is now `Promise<String>`; `eval_critical` awaits it. No
    `RefCell` borrow is held across the await.

- **Allowlist enforcement** (`enforceFetchAllow`, applied synchronously in `__hostFetch`):
  - `config.fetch === false` (or unset) → **block all** → typed `FetchBlockedError`.
  - `config.fetch === true` → allow all.
  - `config.fetch === [hostnames]` → allow only those hostnames (case-insensitive); anything
    else → typed `FetchBlockedError`.
  - Non-http(s) protocols and unparseable URLs → typed `FetchBlockedError`.
  - A blocked request surfaces as a **rejected VM promise** (so `await host.fetch(...)`
    throws a typed error inside the cell, caught and returned as `{ok:false, error}`); the
    **socket stays alive and the kernel remains usable**.

- **Determinism is preserved.** Fetch is an **explicit host effect**: `__hostFetch` never
  calls `ctx.entropy`, so the seeded clock/RNG counters are untouched. Verified live: a cell
  that fetched reported `clockCalls:2` (only the rebind/Date calls, fetch added zero), and two
  fresh seeded kernels produce **byte-identical** `[Date.now(), Date.now(), Math.random()]`
  sequences. The async pump's `evalCode("__cellP")` reads do not call the seeded clock.

- **Safety caps.** Response body capped at 2 MB text; ≤64 headers copied. (Keeps a giant
  response from blowing the heap / snapshot envelope.)

- **globalOutbound posture:** outbound requests use the DO's own `fetch()` (standard workerd
  egress). No new bindings; nothing widens the worker's network surface beyond what the
  allowlist permits per-session.

## P3.2 — Error-as-VALUE preview

Before: a cell that *returned* (not threw) an Error previewed as `{}` because Error fields
(`name`/`message`/`stack`) are non-enumerable, so `JSON.stringify(dump)` dropped them.

V0.3 `_preview` detects an Error result (`handle.isError`, or a constructor name ending in
`Error`) and surfaces:
- `valueType: "error"`,
- `valuePreview`: `"<name>: <message>\n<short stack (≤4 frames)>"`,
- `value`: the structured `{ name, message, stack }` (so callers can read `value.message`).

Thrown errors keep their existing `{ok:false, error:{name,message,stack}}` path (BUG-1).

## Accepted limits / gotchas
- The async pump re-arms the interrupt-tick budget around each microtask drain, so a runaway
  loop in a **post-await continuation** is still preempted. The loop-preemption invariant from
  V0.2 (default 1200 / cap 2000 ticks) is unchanged and verified.
- Response body is capped at 2 MB text; binary bodies are returned as text.
- On a brand-new worker version the very first WS connect + create + fetch can exceed a tight
  RPC timeout purely from cold start; the smoke warms the DO with a `gen` call first.
