## Summary

V0.1 makes the durable QuickJS REPL a **usable dynamically-configured stateful JS env**.
All six V0 bugs are fixed and the V0.1 features (dynamic config persisted across cold
wake, host tools, output capture, structured value preview, client SDK) are implemented
and verified live. Core V0 durability is preserved (eval → evict → restore, SQLite-first
snapshot, atomic checkpoint, seeded determinism, engine-hash + size-admission guards).

Live smoke (`v0.1/smoke.mjs`): **24/24 checks passed.**

## Bug fixes (verified)

| Bug | Fix | Evidence |
|---|---|---|
| BUG-1 (BLOCKER) eval-mutex deadlock on any uncaught error | `glue.js evalCode` catches the QuickJS exception, returns JSON `{ok:false,error:{name,message,stack}}`; never throws across the wasm boundary so `lib.rs eval_critical` always releases the mutex | `throw`/`ReferenceError`/`TypeError`/`SyntaxError` each return `ok:false` + typed error; the **next eval works** (`x+1 == 42` after a throw); `reset` recovers a thrown session |
| BUG-2 + BUG-4 live image never reclaimed | `dump()` runs `kernel.runGC()` before snapshot; size guard on post-GC bytes | `sizeGz` shrinks after `delete globalThis.big`; session keeps checkpointing (`store=sqlite`) after a free |
| BUG-3 no CPU/wall budget; runaway kills DO (WS 1006) | per-cell `interruptHandler` tick budget (`cellBudgetTicks`, default 30000) + secondary wall-clock; typed `TimeoutError` | `while(true){}` → `TimeoutError` in ~1.4s, **socket stays open (readyState=1)**, next eval works |
| BUG-5 `performance.now()` unseeded | seeded off the same clock counter as `Date.now()` | control vs restored sequence `[Date.now,Math.random,performance.now,...]` **byte-identical** across dump/restore |
| BUG-6 `restoreSource` mislabels R2 restores | `lib.rs ensure_glue()` emits `r2-restore`/`sqlite-restore` from the branch that fetched bytes | restore label correctly `sqlite-restore` (R2 branch emits `r2-restore`) |

## V0.1 features (verified)

- **Dynamic config + persistence across cold wake.** `{t:"create", config:{clock:"seeded",
  rngSeed,capture,cellBudgetMs,cellBudgetTicks,tools}}` persists to the DO `meta` row.
  After a `{t:"evict"}` (in-memory kernel dropped), the next eval cold-restores with the
  **same seeded clock** and **re-registered host tools** — `host.echo('post-restore')`
  works and namespace state (`survivor`) survives (`survivor+1 == 8`, `restoreSource=sqlite-restore`).
- **Host tools.** `host.echo(...)`, `host.add(20,22)==42`, `host.kv.put/get` all work via
  the `__hostCall(name, jsonArgs)` boundary (recursive proxy supports `host.kv.put`).
- **Output capture.** `console.log('hi',{a:1})` → `logs:[{level:"log",text:"[\"hi\",{\"a\":1}]"}]`.
- **Value preview.** numbers/strings return native `value`; objects return structured
  `value` + `valuePreview` + `valueType` (e.g. `{k:'v',n:[1,2]}` → `valueType:"object"`).
- **Client SDK** (`sdk/index.mjs`). `connect/eval/reset/gen/create/evict/close`,
  reconnect-safe, re-sends config on reconnect. Validated live (eval, host tools, log
  capture, error handling, evict+restore durability all pass).

## Protocol (final)

```
create  {t:"create", config:{clock,rngSeed,capture,cellBudgetMs,cellBudgetTicks,fetch,tools}}
            -> {ok, config, restoreSource}
eval    {t:"eval", src, config?}
            -> {ok, value, valuePreview, valueType, logs:[{level,text}], error?:{name,message,stack},
                cell, restoreSource, checkpoint}
reset   {t:"reset"} -> {ok, epoch}            (recovers a wedged/errored session)
gen     {t:"gen"}   -> {generation, inMemory, epoch, committedCell, engineHash}
evict   {t:"evict"} -> {ok, droppedInMemory}  (test hook)
```
Config persisted to the DO `meta` row; restored + host tools re-registered before the
first eval on cold wake.

## Notes / open issues
- On workerd the QuickJS interrupt handler is sparse for a *truly empty* `while(true){}`;
  the tick budget reliably preempts any loop that touches a value/property/call (the
  common case) and the default trip time is a few seconds, safely under the ~30s DO wall
  limit. `cellBudgetTicks` is configurable + capped at 200000.
- The R2-overflow GC-reclaim path could not be exercised live because the seeded RNG
  compresses well (a 3MB seeded-random buffer gzipped to ~121KB, under the 2MB-gz
  threshold); the pre-dump `runGC()` reclaim is proven (image shrinks after free) and the
  R2 overflow path itself is the unchanged V0-verified one.
- `fetch` capability is plumbed through config but outbound network egress from the VM is
  reserved for a later iteration.
