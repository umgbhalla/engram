## 1. Is the in-cell alloc-bomb gap CLOSED?

**YES — for the realistic JS-bytecode allocation shapes it was designed for.** Partial for native-extension single-allocation shapes (one new gap, see below).

The QuickJS interrupt handler (already firing ~per 20k bytecode ops alongside the 1200-tick loop budget) now, on **every** invocation while a user cell is in flight (`_memWatch` armed in `evalCode`), reads the WASM linear-memory buffer `byteLength` via `this.kernel._getMemory().buffer.byteLength`. It captures the buffer size at cell entry (`_memBufBase`) and trips on **per-cell growth** (`buf - base > 8 MB`) or an **absolute backstop** (`buf > 16 MB`), whichever first — both below the 18 MB (`MAX_DUMP_BUFFER_BYTES`) dump ceiling, so the cell aborts before the buffer becomes undumpable. On trip it raises the interrupt and `_formatError` labels the unwound cell a typed **`MemoryLimitError`** → `{ok:false,error}`, socket alive, namespace usable, post-abort checkpoint still dumps cleanly (`store:sqlite`).

**Per-case (numeric, observed):**

| Case | Shape | Result | Detail |
|---|---|---|---|
| 1 | push-loop array `a.push(new Array(256))` | **PASS (4/4)** | typed `MemoryLimitError` at buffer ~17.4 MB, reply returns in ~0.4 s (no hang), socket alive (`1+1=2`), pre-bomb namespace (`S='surv'`, `N=9`) intact. v0.7 hang **CLOSED**. |
| 2c | huge zod tree `for 500k: s=z.array(s)` | **PASS** | `MemoryLimitError` at ~17.7 MB in ~3.9 s, socket alive, `alive:42` intact. (The lone "HUNG" batch line was a harness race probing too soon, not a kernel failure.) |
| 3 | legit 12 MB Uint8Array, touched/summed | **PASS** | completes `ok=true` value=10752, checkpoints, no false trip. |
| 2a | huge string doubling `s=s+s` | **CONTAINED, different error** | trips QuickJS `InternalError: string too long` before buffer hits the limit; socket alive, namespace intact. Acceptable. |
| 2b | `structuredClone` of a 100k-elem array, 1e6 loop | **FAIL — WS 1006** | single oversized **native-C** allocation grows buffer from <16 MB into uncatchable Error 1102 between two interrupt callbacks. New gap. |

**Trip timing:** fires at buffer ~17.4 MB (one ~20k-op overshoot past the 16 MB line — bounded as designed), ~325–561 ms, 6/6 reliable across repeated runs. Per-cell growth (not absolute-only) is required because the WASM buffer is monotonic and never shrinks — an absolute-only limit would wedge every cell after the first big one.

**Probe choice:** uses the plain `buffer.byteLength` ArrayBuffer read (re-entrant-safe), NOT `getMemoryUsage()`, which returned **0** when called re-entrantly from inside the interrupt. Cost: ≤~1200 cheap reads/cell, under the ~1.6k/turn workerd throttle.

**Fundamental limit:** the tripwire is bytecode-interrupt bound. Allocation spread across bytecode iterations is caught; a single oversized native-C allocation (a `.so` extension call, e.g. one big `structuredClone`, or `new Uint8Array(1MB).fill()`) yields no bytecode between callbacks and can outrun it — same root-cause class as v0.7 risk #1, now newly reachable via the Tier-0 native extensions. QuickJS's own out-of-memory stays catchable (socket alive).

**Functional evidence (live):** `crypto.subtle.digest('SHA-256', encode('abc'))` = `ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad` (correct real WebCrypto, identical after restore); `TextEncoder` round-trip `'héllo★'` lossless; `URL().hostname` + query parsing correct; `structuredClone` deep object + Map preserved with deep-copy independence; `Headers` set/get/append/has case-insensitive.

**Restore survival:** every cold wake reported `restoreSource=sqlite-restore` (genuine heap-blit, not fresh) and all five natives re-provided on the new instance. Mechanism: `glue.js buildExtensionDescriptors()` builds the `ExtensionDescriptor[]` in canonical `EXTENSION_ORDER` (each a precompiled `WebAssembly.Module`) and passes the **same array** to BOTH `QuickJS.create` AND `QuickJS.restore`. Native bindings (function-table entries at fixed memory/table bases) are not in the heap blit, so `restore()` re-instantiates each extension at the snapshot-recorded base **before** overwriting linear memory — requiring identical descriptors in identical order, guaranteed by the single source.

**Determinism: NOT regressed.** Two fresh seeded kernels produce byte-identical `Date.now`/`Math.random`/seeded `getRandomValues`. Seeded `crypto.getRandomValues` is byte-identical across two independent evict+restore cycles (`231,199,71,87,51,91,94,167` both times) — the RNG routes through the already-seeded WASI `random_get` (mulberry32), replaying through restore; a seeded JS shim is layered on top as belt-and-suspenders. `crypto.subtle.digest` is real but entropy-free (deterministic over input), so it does not break replay. Gated on `config.clock` as before.

### New env fidelity the VM now has
The VM gained real **WebCrypto** (`crypto.subtle` digest/sign/verify, `getRandomValues`, `randomUUID`), **TextEncoder/TextDecoder**, **URL/URLSearchParams**, **structuredClone**, and **Headers** — five web-platform globals previously absent — all backed by native `.so` (renamed `.wasm`) shared libraries rather than JS shims, all surviving cold restore, all determinism-safe under seeded sessions.

## 4. Bugs / risks (ranked)

1. **[Medium — new] Single oversized native-extension allocation WS-1006s the DO (case 2b).** One `structuredClone` of a very large object allocates a big block inside native C where the bytecode interrupt never fires, jumping <16 MB → uncatchable Error 1102 between callbacks. Recoverable on reconnect, but kills the in-flight socket. Mitigation: cap `structuredClone` input size, or add a used-heap check at the native-call boundary.
2. **[Low — pre-existing] Native-C-builtin bombs outrun the tripwire generally** (e.g. `new Uint8Array(1MB).fill()` loop). Same fundamental ceiling as the tick-budget loop preempt; OOMs catchably or grows under inner try/catch. Realistic JS-loop alloc bombs are caught.
3. **[Low — pre-existing v0.7 #3] `crypto.subtle.digest` promise-preview shows `{}`.** The digest works (verified via captured console log); only the async-value preview is lossy, by design.
4. **[Low — by design] huge-string `s=s+s` trips `InternalError` not `MemoryLimitError`.** Fully contained (socket alive, namespace intact), just via the engine's string-length cap rather than the tripwire.
5. **[Note] `src/ext/` is gitignored** (derived from `node_modules` at build by `scripts/prepare-extensions.mjs`, like `src/quickjs.wasm`). Ext `.so` MUST be renamed `.wasm` — wrangler's esbuild maps the `CompiledWasm` loader by extension and errors `No loader is configured for .so files` otherwise (bytes unchanged).
