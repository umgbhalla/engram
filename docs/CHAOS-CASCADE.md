PROSE:

The cluster's central invariant is that **the WASM linear memory buffer is the durable state primitive**: `engine/src/lib.rs` exports a C ABI (`snapshot`, `restore`, `eval_cell`) that lets the Rust DO shell in `apps/kernel/src/lib.rs` dump and blit raw bytes into SQLite (chunked at ~64 KB rows, R2 overflow above ~2 MB gzip), so a session that hibernates mid-namespace resumes with closures, promises, and typed arrays intact — no source replay. The JS glue (`kernel-glue.ts`) owns the WASM instantiation boundary: it constructs all WASI imports (clock, RNG, fd_write — all seeded/deterministic), injects stdlib modules exactly once at `create` time (they survive in the snapshot), routes `host.*` calls from the VM back to the DO via a registered callback table, and enforces SSRF guards on outbound fetch before any network crosses the isolate boundary. `repl-transform.ts` sits at the entry point of every `eval` call and rewrites depth-0 `let`/`const`/`class` declarations into `globalThis` assignments and hoists named functions, so the QuickJS heap accumulates namespace across cells the way a Jupyter kernel does. The cluster couples upward to the supervisor (`apps/cloud`) which routes per-tenant WebSocket frames to the correct DO instance, and downward to the R2 bucket and the Worker-Loader registry for Tier-1 hash-worker invocation; it has no direct dependency on the UI or SDK layers — those speak only the JSON wire protocol this cluster exposes over WebSocket.

PROSE:

The cluster's central invariant is the **staged-commit coherence contract**: all host-mediated state (fs writes, kv mutations, timer seeds, fetch results) is staged in-memory during a cell's execution and flushed atomically together with the QuickJS WASM heap snapshot to R2/SQLite, so a cold restore is always byte-identical with zero torn writes. The **unified `/workspace` filesystem** enforces a single R2-backed namespace (`fs/<doId>/<path>`) across four surfaces — the in-heap VFS used by QuickJS `fs.*` calls, the hash-worker's `env.VFS` gateway, the sandbox container's FUSE mount, and any artifact pointer — with the SQLite manifest as the sole coherence authority and the in-heap cache demoted to a warm-path acceleration layer only. The **muscle ladder** is capability-injected fail-closed: the kernel DO holds the `LOADER` binding for Tier-1 content-addressed Worker-Loader isolates and the `ENGRAM_SANDBOX_URL`+`KEY` secret for Tier-2 Linux container calls; neither tier is reachable without the injected credential, and neither tier can reach outside its VFS prefix without an explicit capability grant. The **net.js socket shim** (Tier-0 stdlib) mediates all TCP/TLS outbound traffic from within the QuickJS VM through a sequential host-call pump — write-drain then read-drain — keeping the single-threaded DO boundary intact and handling stale-socket wakeups safely. Coupling to other clusters: the kernel's WASM snapshot mechanism (heap-blit + globals + tables) is the primitive that makes all of this durable — without it the muscle ladder would be stateless; the SDK and cloud supervisor clusters consume the kernel's `/eval` and `/ws` interfaces but are unaware of which muscle tier executed a given cell.

The `apps/cloud` supervisor DO is the multi-tenant control plane: it receives HTTP/WebSocket requests, validates WorkOS JWTs to resolve org and user identity, enforces role-based per-op permissions (eval/snapshot/fetch/etc.), and routes each session to a `ctx.facets.get(sessionId)` kernel facet that owns its own isolated SQLite and Rust-QuickJS heap snapshot. ## 3. Load-bearing invariants (the whole system rests on these)

1. **Heap-blit durability.** `memory.buffer` (+ globals + tables) round-trips through SQLite/R2 byte-faithfully. Not logical serialization — closures/promises/object graphs survive with no source replay. Break this → everything above is stateless garbage.
2. **Staged-commit coherence.** All host-mediated mutations in a cell (fs writes, kv, timer seeds, fetch results) are staged in-memory and flushed **atomically with the heap snapshot**. Cold restore is always byte-identical with zero torn writes. Crash-atomic via chunk-count manifest guard + swap-then-delete R2 path. (No raw `BEGIN/COMMIT` on DO SQLite — workerd write-coalescing only.)
3. **W4 chain reconstructs exactly.** Committed snapshot = one W5-compacted full base + ≤20 per-cell byte-deltas. `restoreW4` recomputes CRC32 of the reconstructed full image and **rejects on mismatch**. A `BASE_EVERY=20` cap bounds chain length; `dumpW4` may also downgrade to a full base on dense mutation.
4. **Size-admission gates on `used_heap`, not raw buffer.** Snapshot/restore guards admit on QuickJS `getMemoryUsage().memoryUsedSize` (manifest `used_heap` column) — so a spike-then-free session checkpoints and cold-restores. The documented dump ceiling (~18MB) clean-rejects (typed `SizeAdmissionError`, socket alive); >~45MB raw buffer fails safe. **This guard is the single critical coupling between the memory subsystem and durability** — the adversarial red-team breach was exactly its absence in the W4 variant.
5. **Triple liveness guard keeps the DO socket alive on adversarial input.** (a) interrupt-tick budget (config default ~1200, cap ~2000) — *must sit below the workerd interrupt-throttle floor* (~1.6k invocations/turn) or infinite loops escape to WS-1006; (b) per-cell buffer-growth tripwire (grow-cap pages checked at eval-begin, catches non-looping fast-array bombs that fire no interrupts); (c) mid-cell used-heap tripwire (typed `MemoryLimitError` on >8MB per-cell growth). Resume does **not** refill the budget (continuation runs under the parking cell's remaining envelope).
6. **Tier-0 native extensions re-wire before the blit.** On restore the 5 `.so` natives are re-instantiated at **fixed bases in the same order** before the heap is blitted, or the heap's internal pointers into them are invalid. Order is load-bearing.
7. **Unified `/workspace` is one R2 namespace, SQLite manifest is sole authority.** Cell `fs.*`, hash-worker `env.VFS`, and container FUSE all resolve to the same `fs/<doId>/<path>` R2 object. In-heap cache is a warm-path accelerator only. Body bytes go to R2 **first**, then the `fs_files` meta row (write-ordering rule).
8. **Determinism externalized.** Clock/RNG/crypto.getRandomValues/randomUUID route through seeded WASI `random_get`/clock; fetch adds zero entropy. Seeded sessions stay byte-identical across restore. setInterval is a no-op; no timer spans a snapshot.

## 4. Trust / capability boundaries (what crosses the I/O boundary)

- **VM heap ⟷ DO host (the WASI/import boundary).** All non-determinism (clock, RNG, fd_write, fetch results) crosses here, seeded/mediated. The VM cannot reach network/disk/time except via registered `host.*` callbacks routed back to the Rust DO. **Fail-closed.**
- **Outbound fetch / sockets (SSRF boundary).** `host.fetch` → DO-side `fetch()`; raw TCP/TLS → `host-sockets.mjs` pump. SSRF guard (mirrored in glue `isBlockedSsrfHost` + `host-sockets.ssrfReason`) blocks loopback/link-local/metadata/private IPs **regardless of caller**. `config.fetch` allowlist (false/true/[hosts]) narrows further; block → typed `FetchBlockedError`/`SSRFBlockedError`, socket stays alive.
- **Muscle ladder (capability-injected, fail-closed).** Tier-1 hash-workers run in fresh isolates with `globalOutbound:null` and a **prefix-scoped** `env.VFS` — cannot escape their R2 prefix without grant. Tier-2 sandbox reachable only via the `ENGRAM_SANDBOX_URL`+`KEY` secret held by the DO; container shares the R2 workspace but reaches the kernel only by writing canonical R2 keys. Neither tier reachable without the injected credential.
- **Multi-tenant auth boundary (supervisor).** WorkOS credential validation resolves org→account; 7-slug per-op permission check (`permissions.includes(slug)`) gates eval/snapshot/fetch/etc. **before** the facet hears the frame. W1-era keys with no permissions claim fall back to the developer profile (documented). Facet SQLite is isolated — a facet cannot read the supervisor's secret or another facet's store. Cross-tenant data leakage validated at zero.
- **Client ⟷ kernel (wire boundary).** SDK/CLI/UI hold **no authoritative namespace state** — only the JSON WS protocol. A reconnecting client reattaches to the same DO id; in cloud topology the supervisor proxies via POST /frame (facet-held client sockets are broken).

**Chaos-test pressure points (blunt):** (1) the `used_heap` admission guard — every memory attack reduces to whether this fires correctly; (2) the interrupt-tick budget vs. workerd throttle floor — the one place an infinite loop can still WS-1006 the DO; (3) native-builtin alloc bombs (`.fill`) that fire no bytecode interrupts; (4) Tier-0 `.so` re-wire ordering on restore (pointer corruption if reordered); (5) staged-fs write-ordering and the container-direct-R2 / no-`fs_files`-row read fallback (coherence edge); (6) supervisor shard as the multi-tenant scale bottleneck; (7) monotonic linear memory — the ~256MB grown-past-envelope session that cannot recover.

## Layer 5 — 8 Chaos Ideas (model: fable)

### Chaos 1. Codec-flip CRC collision on the W4 delta chain  `[session]`

- **Target invariant:** Heap-blit durability + W4 chain reconstructs exactly (CRC32 guard)
- **Attack:** Drive a session so consecutive cells alternate dense vs. sparse heap mutation, forcing dumpW4 to oscillate between full-base downgrade and byte-delta, while the per-chunk codec auto-switches gzip->zstd at the size threshold. Time a genuine eviction to land exactly on the BASE_EVERY=20 rollover, then submit one more delta whose changed bytes sit on the chunk boundary where the codec just flipped. Cold-restore and check whether restoreW4 reconstructs the same image the live VM had.
- **Why it might break:** The CRC32 guard validates the reconstructed full image, but the base/delta downgrade decision and the per-chunk codec choice are taken independently at dump time; an eviction landing between the base-write and the final delta-commit can leave an orphaned base whose chunks were encoded under a codec the reconstruction path assumes differently. CRC may pass on a structurally-correct-but-stale image, silently resurrecting a prior generation's heap with no replay to correct it.

### Chaos 2. High-water raw-image restore stampede  `[host]`

- **Target invariant:** Size-admission gates on used_heap, not raw buffer; WASM monotonic memory
- **Attack:** In N sessions in parallel, spike each heap to ~40MB raw, free everything, run runGC + scrub_arena so used_heap drops under 18MB and gzip shrinks the stored image, checkpoint successfully. Then force all N sessions to evict and cold-restore simultaneously (kill keep-warm, idle them out together).
- **Why it might break:** Admission on cold restore reads the recorded used_heap (small) and admits, but the raw image the WASM instance must allocate is still pinned at the ~40MB monotonic high-water mark. One restore is fine; a synchronized stampede of high-water restores forces every isolate on the placement to allocate tens of MB at once, and the per-restore guard never accounts for concurrent peak — classic admit-locally-OOM-globally.

### Chaos 3. Pre-load the interrupt throttle, then loop  `[session]`

- **Target invariant:** Interrupt-tick budget must sit below the workerd interrupt-throttle floor
- **Attack:** Saturate one DO turn with many host.fetch / host-call parks and resumes to push workerd's host-interrupt-callback invocation count toward its throttle threshold within the turn, THEN in the same turn submit a tight property-store infinite loop (`while(true){o.a=1}`).
- **Why it might break:** The 1200/2000 tick budget was certified against a fresh-turn throttle floor (~1.6k invocations). Prior in-turn load is documented as load-sensitive and lowers that floor; if the floor drops below 1200 the interrupt callback gets throttled before the budget trips, so the loop runs uninterrupted and OOM/1006-kills the DO socket with no typed TimeoutError. The guard's safety margin is exactly the thing the attacker shrinks.

### Chaos 4. Single-syscall native alloc bomb under the tripwire  `[session]`

- **Target invariant:** Mid-cell used-heap tripwire (>8MB per-cell growth)
- **Attack:** Issue one native-builtin allocation that grows linear memory past 8MB in a single C call that emits zero bytecode interrupts: e.g. `new Uint8Array(60e6).fill(7)`, a giant `String.prototype.repeat`, or a `structuredClone` of a self-referential megastructure routed through the Tier-0 native .so. No loop, no eval boundary.
- **Why it might break:** Both the tick budget and the mid-cell tripwire are sampled on bytecode-instruction interrupts; a single native builtin that allocates tens of MB returns to the interpreter only after it has already grown monotonic memory past the dump ceiling. ### Chaos 5. Supervisor decapitation during facet keep-warm  `[supervisor]`

- **Why it might break:** Facets have no alarms, so the supervisor is the single owner of every facet's keep-warm and TTL timer. - **Target invariant:** Unified /workspace coherence: R2 body-first, fs_files meta row is sole authority

### Chaos 8. Parked-continuation resume across a generation boundary  `[session]`

Premise correction: I no longer claim a codec flip or independent codec choice. Uniform zstd, per-row codecs, and the atomic synchronous txn() are all conceded. I instead attack the path the CRC failure ROUTES INTO — the oplog-replay fallback, which has NO CRC gate.

Setup (drive a session past one BASE_EVERY boundary with a >8MB used heap so the base spills to R2):
1. Run cells 0..18 that build durable in-heap state across the delta chain (each cell mutates a large object/closure/promise graph). delta_seq climbs toward BASE_EVERY=20. Keep usedHeap > SQLITE_HOT_MAX (8MB) so the NEXT full base is forced to R2, not SQLite.
2. On the cell where `prev.delta_seq + 1 >= 20`, checkpoint_impl forces a FULL BASE. The full-base path (lib.rs ~3055-3128) does, in order: `bucket.put(&new_key, bytes).await` (the new base to R2), then a synchronous txn() that DELETEs snap_chunks/delta_chunks/**oplog**/snap_manifest, **re-seeds oplog with seq=0 = ONLY this one cell** (line 3097), writes the new manifest pointing at the fresh R2 key with snap_mode="full"/delta_seq=0, then commit_dump().

R2 is read-after-write eventually consistent for a brand-new key across region/edge failover; under the propagation gap the GET returns **None (404 durable miss)**. r2_get_resilient classifies a durable miss as NOT transient (lib.rs line 2092), returns R2Read::Missing, and sets r2_replay_reason="r2-missing-replay" — it does NOT retry, does NOT open the breaker, does NOT fail.
5. restore() then takes the `is_r2 && gz.is_none()` branch (lib.rs ~2792): it calls `read_oplog()` → `replayJournal`. But the oplog was just truncated at the base rollover to a SINGLE cell (step 2). 6. Crucially, the replay path NEVER checks final_crc.
