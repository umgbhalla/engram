# V0.1 — Dynamically-configured stateful JS environment

> Goal (user): "a dynamically configured JS environment which remains as a stateful REPL."
> Make the V0 kernel actually **usable and useful** as a single-tenant product surface.
> Facets/multi-tenancy deferred to V1 (see ADR-0003) — only if V0.1 proves out.

## Thesis
V0 proved the *substrate* (durable live-heap REPL, fast cold wake). V0.1 makes it a
**configurable environment**: a session boots with a declared capability set + host
tools, evals cells against a persisted live namespace, and survives hibernation — the
config travels with the snapshot so the environment is identical on cold wake.

## Scope

### 1. Dynamic configuration (the "dynamically configured" part)
A session is created with a **config** that shapes the JS environment, injected at the
host-import boundary (NOT yet via Worker Loader / facets):
- **Capabilities:** outbound `fetch` on/off (and allow-list), clock mode (real vs seeded
  deterministic), RNG seed, console capture on/off, CPU/wall budget per cell.
- **Host tools:** named host functions exposed into the VM as `host.<name>(...)` via the
  `host_call(name, args)` boundary — e.g. `kv.get/put`, `tool.<x>`, agent tool-calls.
- Config is **persisted in the snapshot manifest** (meta row) so a cold-wake restore
  re-establishes the exact same environment + re-registers the same host callbacks.

### 2. Usability surface
- **Output capture:** `console.log/warn/error` buffered per cell, returned with the result.
- **Result serialization:** structured-clone-ish value preview (not just stringify);
  errors returned with `name/message/stack`, kernel stays usable after a throw.
- **Cell ergonomics:** cell ids/sequence, last-value (`_`), multi-statement cells,
  top-level `await` in a cell.
- **Client SDK:** a tiny TS/JS client (`connect(url, {id, config})` → `eval(src)` /
  `reset()` / `gen()`), wrapping the WS protocol. Reconnect-safe.

### 3. Hardening (from V0 review + stress results)
- Confirm/repair anything the stress suite (`wb2xbvb46`) surfaces.
- Long-loop / CPU-budget: a cell that runs away must be killed cleanly (typed timeout),
  not crash the DO (WS 1006). Enforce a wall-clock budget around eval.
- Validate the R2-overflow path under real replace timing (open issue).
- Keep all V0 guards: size-admission, engine-hash, atomic checkpoint, determinism.

### Out of scope for V0.1 (→ V1)
- Facets / supervisor / multi-tenant (ADR-0003).
- Per-tenant kernel code via Worker Loader.
- Python kernel (RustPython).
- Streaming gunzip for >30 MB images.

## Protocol delta (additive to V0)
```
create  {t:"create", config:{ fetch:bool|string[], clock:"seeded"|"real",
                               rngSeed:int, capture:bool, cellBudgetMs:int,
                               tools:[name...] }}
eval    {t:"eval", src} -> { ok, value, valuePreview, logs:[...], error?, cell, restoreSource }
gen / reset / evict      (unchanged)
```
Config persisted to `meta`; restored before first eval on cold wake.

## Acceptance (is it usable/useful?)
- Drive a real multi-cell session: define fns, fetch (if enabled) through the host
  boundary, console output captured, errors clean, state survives an eviction mid-session.
- A demo "agent code-mode" loop: host tool-calls from inside the VM, results returned,
  session resumes after idle with tool state intact.
- p50 cold wake still sub-second; no DO crash on any cell input.

If V0.1 demonstrably works as a useful configurable stateful JS env → proceed to V1 facets.
